Chapter 2 Data Manipulation

We will discuss data transformation using data for all flights that departed NYC (i.e.Â JFK, LGA or EWR) in 2013. This data is contained in the flights data frame that comes from the nycflights13 package. Before beginning, install this package and load it (using nycflights13).

There are five key dplyr functions that allow you to solve the vast majority of your data manipulation challenges:

  1. Pick observations by their values (filter()).
  2. Reorder the rows (arrange()).
  3. Pick variables by their names (select()).
  4. Create new variables with functions of existing variables (mutate()).
  5. Collapse many values down to a single summary (summarise()). This is often used in conjunction with group_by(), which changes the scope of each function from operating on the entire dataset to operating on it group-by-group.

2.1 Filter

Suppose we want to look at just morning flights (before 10am).

morning_df = filter(flights, hour < 10)

How about morning flights in January.

morning_df = filter(flights, hour < 10, month == 1)

2.2 Arrange

What are the attributes of the flights with the biggest departure delays?

arrange(flights, desc(dep_delay))

2.3 Select

Suppose we donât care about the flight number.

arrange(flights, !flight)

2.4 Mutate

Suppose we want to add a speed variable.

mutate(flights, speed = distance/air_time * 60)

2.5 Group-By/ Summarise

Suppose we want to know the mean departure delay by airline.

flights %>% 
  group_by(carrier) %>% 
  summarize(mean_delay = mean(dep_delay, na.rm = T), n = n()) %>% 
  arrange(desc(mean_delay))
